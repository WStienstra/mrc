# MRC — Mandelbrot Reference Coding

A lossless compression codec for short numeric byte-strings whose codebook
is **regenerated by both encoder and decoder from a few bytes of parameters**,
not transmitted. The reference set is built from Mandelbrot and Julia escape-time
orbits at multiple zoom levels; the encoder stores `(atlas address + small
affine fit + LZMA-compressed correction residual)` per block. Decoding is
deterministic and bit-exact.

The interesting regime is the **sub-kilobyte payload** band where general
purpose codecs (gzip, LZMA, zstd) cannot amortise their own framing cost.
On real seismometer and machine-bearing telemetry the variable-block variant
(`MRC v2.3`) consistently beats LZMA-9-extreme on 256 B / 512 B / 1 KB slices.

This repository contains the reference Python implementation, a Rust
prototype, the paper, and the benchmark fixtures used for the headline
results.

## Headline results (`bench/realbench-results.txt`)

Real ground-vibration telemetry (seismometer, 40 Hz / 100 Hz) and CWRU
machine-bearing accelerometer, 5 random offsets per (source × slice size),
versus the best of `gzip-9`, `LZMA-9-extreme` and a hand-rolled
`int16 delta + zigzag + plane-split + LZMA` baseline.

## Reproducing

```bash
git clone https://github.com/WStienstra/mrc.git
cd mrc
bash reproduce.sh
```

`reproduce.sh` installs the two Python deps (`numpy`, `matplotlib`) and
re-runs the benchmark. The original results are committed for diffing.

## Wire format

Container magic bytes:

- `WFC2` — v2.3 fixed-block stream (`src/mrc.py`). Historic name, kept for
  compatibility with already-encoded files.
- `MRC3` — variable-block stream, atlas 0 implied (`src/mrc3.py`).
- `MRC4` — variable-block stream, explicit atlas ID byte (`src/mrc3.py`).

After the magic byte and a small header, the payload is four
LZMA-raw streams: `[indices][scales][offsets][residuals]`. Decoders
regenerate the atlas locally from the atlas ID; nothing about the codebook
is on the wire.

## Status

This is a research codec, not a production library. The Python reference is
slow (pure NumPy, no SIMD); the Rust side only ships the atlas generator.
There is no public API stability guarantee. The benchmark fixtures, the
paper, and `reproduce.sh` are the artefacts intended for review.
